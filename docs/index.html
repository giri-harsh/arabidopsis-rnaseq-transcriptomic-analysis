From raw counts to significant genes, with the statistics shown, not assumed.

A reproducible command-line pipeline that takes a gene × sample RNA-seq count matrix from public Arabidopsis thaliana abiotic-stress experiments through normalization, negative-binomial dispersion modeling, Wald testing, and Benjamini–Hochberg correction — producing the same candidate DEG list a wet-lab collaborator could act on.

01 — Overview

What this pipeline actually does

This repository implements a complete differential gene expression (DGE) workflow for RNA-seq count data, targeting abiotic stress-response studies in Arabidopsis thaliana. Every stage — filtering, normalization, dispersion estimation, hypothesis testing, FDR correction, clustering, and figure generation — is implemented as real, tested Python code, not a notebook sketch. The pipeline is dataset-agnostic: any two-condition raw count matrix with matching sample metadata can be analyzed.

02 — Motivation

Why build this

Public repositories like GEO and SRA hold thousands of plant stress-response RNA-seq datasets, but turning a raw count matrix into a defensible gene list requires correctly chaining normalization, overdispersion modeling, and multiple-testing correction — steps that silently produce wrong answers if done naively (e.g. comparing raw counts across samples of different sequencing depth, or applying a t-test where an NB-GLM Wald test belongs). This project exists to implement that chain correctly and transparently.

03 — Objectives

What the analysis sets out to answer

Identify genes whose expression differs significantly between control and stress conditions, beyond normal biological variability.

Quantify effect size (log2 fold change) alongside statistical confidence (adjusted p-value) for every tested gene.

Cross-check DE structure against unsupervised methods (PCA, clustering) — do samples separate by condition independently of the test?

Produce a reproducible, auditable trail from raw counts to final figures and reports.

04 — Research Workflow

Pipeline, step by step

This sequence runs in order — each stage consumes the previous stage's output.

01

Load & validate

Read raw counts + sample metadata; check sample alignment, integer counts, missing values.

02

Filter low-count genes

Remove genes with insufficient total signal or too few samples expressing them.

03

Normalize

Median-of-ratios size factors (DESeq2 method) correct for sequencing-depth differences between samples.

04

Estimate dispersion

PyDESeq2 fits per-gene dispersion: gene-wise estimate → trend curve → MAP shrinkage.

05

Fit the model & test

Negative binomial GLM per gene; Wald test on the condition coefficient.

06

Correct for multiple testing

Benjamini–Hochberg FDR correction across all tested genes.

07

Extract significant DEGs

Filter by adjusted p-value and minimum fold-change magnitude.

08

Cross-check & visualize

PCA, hierarchical clustering, KMeans, and the full figure set are generated from the same validated data.

06 — Statistical & ML Methods

How significance is actually determined

Statistical

Normalization: median-of-ratios size factors.

Dispersion: gene-wise → trend → MAP shrinkage (PyDESeq2 / DESeq2 methodology).

Test: Wald test on the fitted log2 fold-change coefficient.

Correction: Benjamini–Hochberg FDR at a configurable alpha (default 0.05).

Machine Learning

PCA: on the top-variance-gene expression matrix — do samples separate along the top components?

Clustering: Ward hierarchical clustering + KMeans, scored against true labels via Adjusted Rand Index.

Dimensionality (optional): t-SNE for qualitative comparison against PCA, perplexity auto-capped for small sample counts.

09 — Key Features

What's actually implemented

  • Real input validation — catches sample mismatches, TPM/FPKM mistaken for raw counts, missing values.
  • DESeq2-methodology dispersion estimation and Wald testing via PyDESeq2 — not an approximation.
  • Benjamini–Hochberg FDR correction applied to every tested gene.
  • PCA, hierarchical clustering, and KMeans as independent cross-checks on DE structure.
  • Every figure exported as both PNG and PDF for publication use.
  • Fully serializable run configuration for reproducibility (run_config.json).
  • Bundled synthetic dataset with seeded ground-truth DEGs so the pipeline runs immediately after cloning.
  • Automated Markdown + CSV/TSV/JSON reporting — no manually-typed numbers.
10 — Biological Insights

Reading the DEG list biologically

The pipeline outputs a statistically validated candidate gene list — genes whose expression shift between conditions exceeds what's expected from replicate variability, at the chosen FDR threshold. Genes with large positive log2 fold change are candidates for stress-induced upregulation (e.g. stress-response transcription factors, osmotic or oxidative stress pathway genes in a real abiotic-stress experiment); large negative fold-change genes are candidates for stress-induced repression of normal growth and metabolic processes. Attaching gene function to these IDs requires cross-referencing a functional annotation resource such as TAIR or Ensembl Plants — this pipeline produces the candidate list, not the functional annotation itself.

11 — Limitations

What this pipeline does not do

  • No GO-term or KEGG pathway enrichment — DEG lists require external biological annotation to interpret function.
  • Small replicate counts (common in public datasets) widen fold-change confidence intervals.
  • t-SNE, if enabled, has no principled variance-explained interpretation.
  • The default ~condition design does not model batch effects unless added as a covariate.
  • Read alignment / quantification (FASTQ → counts) is out of scope; the pipeline starts from an already-quantified count matrix.
12 — Future Work

Where this goes next

  • GO-term / KEGG pathway enrichment on significant DEGs.
  • Multi-factor design support (e.g. ~batch + condition) and interaction contrasts.
  • Automatic outlier-sample detection prior to model fitting.
  • Time-course RNA-seq support via likelihood ratio test workflows.
  • Default integration of a real public GEO/SRA Arabidopsis dataset with automated download.
